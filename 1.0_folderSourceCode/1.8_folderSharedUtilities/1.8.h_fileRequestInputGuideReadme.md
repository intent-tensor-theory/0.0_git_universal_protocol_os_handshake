# Request Input Guide

## What is Request Input?

The **Request Input** section is where you define the actual data you want to send to the API. This can be a cURL command, JSON payload, form data, or file upload depending on what the API expects.

---

## Input Methods

### 1. cURL Command (Universal)

The most flexible option. Paste any cURL command and it will be parsed and executed.

```bash
curl -X POST https://api.example.com/v1/data \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"name": "test", "value": 123}'
```

**cURL Flags Reference:**

| Flag | Meaning | Example |
|------|---------|---------|
| `-X` | HTTP method | `-X POST`, `-X PUT`, `-X DELETE` |
| `-H` | Add header | `-H "Content-Type: application/json"` |
| `-d` | Send data (body) | `-d '{"key": "value"}'` |
| `-F` | Form data / file upload | `-F "file=@photo.jpg"` |
| `-u` | Basic auth credentials | `-u username:password` |
| `-k` | Ignore SSL errors | `-k` |
| `-L` | Follow redirects | `-L` |
| `-v` | Verbose output | `-v` |
| `-o` | Output to file | `-o response.json` |

---

### 2. JSON Payload

For APIs expecting JSON, you can write the payload directly:

```json
{
  "model": "gpt-4",
  "messages": [
    {"role": "user", "content": "Hello!"}
  ],
  "temperature": 0.7
}
```

**Tips:**
- Validate your JSON at jsonlint.com before pasting
- Use double quotes for keys and string values
- No trailing commas after the last item

---

### 3. Form Data

For multipart form submissions:

```
field1=value1
field2=value2
file=@/path/to/file.pdf
```

---

### 4. XML/SOAP

For SOAP APIs:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<soap:Envelope xmlns:soap="http://schemas.xmlsoap.org/soap/envelope/">
  <soap:Body>
    <GetUserRequest>
      <UserId>12345</UserId>
    </GetUserRequest>
  </soap:Body>
</soap:Envelope>
```

## The {INPUT} Placeholder System

Use `{INPUT}` as a placeholder in your request template. When you execute, the system will substitute your actual input value.

**Example Template:**
```bash
curl -X POST https://api.openai.com/v1/chat/completions \
  -H "Authorization: Bearer {API_KEY}" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-4",
    "messages": [{"role": "user", "content": "{INPUT}"}]
  }'
```

**Available Placeholders:**

| Placeholder | Description | Source |
|-------------|-------------|--------|
| `{INPUT}` | Your runtime input | Text field / file |
| `{API_KEY}` | Stored API key | Credentials form |
| `{ACCESS_TOKEN}` | OAuth access token | OAuth flow result |
| `{TIMESTAMP}` | Current Unix timestamp | Auto-generated |
| `{UUID}` | Unique identifier | Auto-generated |
| `{BASE_URL}` | Platform base URL | Platform config |

## HTTP Methods Explained

| Method | Purpose | Has Body? | Idempotent? |
|--------|---------|-----------|-------------|
| `GET` | Retrieve data | No | Yes |
| `POST` | Create new resource | Yes | No |
| `PUT` | Replace entire resource | Yes | Yes |
| `PATCH` | Partial update | Yes | No |
| `DELETE` | Remove resource | Sometimes | Yes |
| `HEAD` | Get headers only | No | Yes |
| `OPTIONS` | Get allowed methods | No | Yes |

| Content-Type | Use For |
|--------------|---------|
| `application/json` | JSON APIs (most common) |

| `multipart/form-data` | File uploads |

| `text/plain` | Plain text |

## File Upload Patterns

### Single File (cURL)
```bash
curl -X POST https://api.example.com/upload \
  -H "Authorization: Bearer TOKEN" \
  -F "file=@document.pdf"
```

### Multiple Files
```bash
curl -X POST https://api.example.com/upload \
  -F "file1=@image1.jpg" \
  -F "file2=@image2.jpg"
```

### File with Metadata
```bash
curl -X POST https://api.example.com/upload \
  -F "file=@document.pdf" \
  -F "description=My document" \
  -F "folder_id=123"
```

## Request Validation

Before execution, the system validates:

1. ✅ Valid URL format
2. ✅ Required headers present
3. ✅ Valid JSON syntax (if JSON body)
4. ✅ File exists (if file upload)
5. ✅ Placeholders have values

**Common Errors:**

| Error | Cause | Fix |
|-------|-------|-----|
| Invalid JSON | Syntax error in body | Check quotes, commas, brackets |
| Missing header | Required header not set | Add `-H "Header: value"` |
| 401 Unauthorized | Invalid/expired token | Refresh credentials |
| 400 Bad Request | Malformed request | Check API docs for format |
| 404 Not Found | Wrong endpoint | Verify URL path |

---

## Best Practices

1. **Start simple** - Test with a basic GET request first
2. **Check the docs** - Every API documents expected request format
3. **Use the right Content-Type** - Mismatched types cause parsing errors
4. **Validate before sending** - Catch errors before they hit the API
5. **Log everything** - Review request/response for debugging
6. **Handle rate limits** - Respect API throttling

## Quick Examples by API

### OpenAI Chat
```bash
curl https://api.openai.com/v1/chat/completions \
  -H "Authorization: Bearer $OPENAI_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"model":"gpt-4","messages":[{"role":"user","content":"Hello"}]}'
```

### GitHub Get User
```bash
curl https://api.github.com/user \
  -H "Authorization: Bearer $GITHUB_TOKEN" \
  -H "Accept: application/vnd.github+json"
```

### Stripe Create Customer
```bash
curl https://api.stripe.com/v1/customers \
  -u sk_test_xxx: \
  -d "email=customer@example.com" \
  -d "name=Test Customer"
```

### Slack Post Message
```bash
curl -X POST https://slack.com/api/chat.postMessage \
  -H "Authorization: Bearer $SLACK_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"channel":"C123","text":"Hello from API!"}'
```
