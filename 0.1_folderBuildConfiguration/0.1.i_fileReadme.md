# Protocol OS - Universal API Handshake

> **Universal API Handshake - Protocol Operating System - Server-Side Architecture**

A comprehensive template for managing multiple authentication protocols and API integrations from a single interface. Built with React + Vite + TypeScript following [Ghostless Coding Architecture](https://medium.com/@intent.tensor.theory/ghostless-coding-architecture-e30465811d8b) principles.

---

## üéØ Overview

Protocol OS provides a unified interface to configure, test, and manage API handshakes across 11 different authentication protocols. Clone this template, add your credentials, and you have a fully functional API management system.

### Supported Protocols

| Protocol | Description | Callback Required |
|----------|-------------|-------------------|
| **cURL Default** | Universal cURL command execution | No |
| **OAuth PKCE** | OAuth 2.0 with Proof Key for Code Exchange | Yes |
| **OAuth Auth Code** | Standard OAuth Authorization Code flow | Yes |
| **OAuth Implicit** | OAuth Implicit Grant for client-side apps | Yes |
| **Client Credentials** | Server-to-server OAuth authentication | No |
| **REST API Key** | Bearer token or API key authentication | No |
| **GraphQL** | Query-based API with strong typing | No |
| **WebSocket** | Real-time bidirectional connections | No |
| **SOAP/XML** | Enterprise SOAP services with WS-Security | No |
| **GitHub Repo Runner** | Repository access and script execution | No |
| **Keyless Scraper** | Zero-auth web scraping and DOM parsing | No |

## üèóÔ∏è Architecture

Protocol OS uses a hierarchical **Platform ‚Üí Resource ‚Üí Handshake** structure:

```
Platform (e.g., "Google APIs")
  ‚îî‚îÄ‚îÄ Resource (e.g., "Calendar API")
        ‚îî‚îÄ‚îÄ Handshake (e.g., "OAuth PKCE + List Events")
              ‚îú‚îÄ‚îÄ Authentication Configuration
              ‚îú‚îÄ‚îÄ Request Model (cURL/JSON/GraphQL)
              ‚îî‚îÄ‚îÄ Execution Results
```

### Serial Number System

Each entity generates a contiguous serial chain:

```
Platform:   PLAT-MASTER-7X4F
Resource:   PLAT-MASTER-7X4F-RES-2B9K
Handshake:  PLAT-MASTER-7X4F-RES-2B9K-HS-M3P1
```

## üíæ Database Providers

Protocol OS supports multiple persistence backends. Toggle between them with a single configuration change:

### localStorage (Default)
No configuration needed. Data persists in browser.

### Supabase
```env
VITE_ACTIVE_DATABASE_PROVIDER=supabase
VITE_SUPABASE_PROJECT_URL=YOUR_SUPABASE_PROJECT_URL
VITE_SUPABASE_ANON_KEY=YOUR_SUPABASE_ANON_KEY
```

### Firebase
```env
VITE_ACTIVE_DATABASE_PROVIDER=firebase
VITE_FIREBASE_API_KEY=YOUR_FIREBASE_API_KEY
VITE_FIREBASE_PROJECT_ID=YOUR_FIREBASE_PROJECT_ID
```

### PostgreSQL
```env
VITE_ACTIVE_DATABASE_PROVIDER=postgresql
VITE_POSTGRESQL_CONNECTION_STRING=YOUR_POSTGRESQL_CONNECTION_STRING
```

### SQLite
```env
VITE_ACTIVE_DATABASE_PROVIDER=sqlite
VITE_SQLITE_DATABASE_PATH=YOUR_SQLITE_DATABASE_PATH
```

## üé® Extending Protocols

Adding a new protocol is straightforward:

### 1. Create Protocol Folder

```
1.3_folderProtocolHandshakeRegistry/
  ‚îî‚îÄ‚îÄ 1.3.12_folderMyNewProtocol/
      ‚îú‚îÄ‚îÄ 1.3.12.a_fileMyNewProtocolHandshakeExecutor.ts
      ‚îú‚îÄ‚îÄ 1.3.12.b_fileMyNewProtocolCredentialFormFields.tsx
      ‚îú‚îÄ‚îÄ 1.3.12.c_fileMyNewProtocolWhitepaperDocumentation.ts
      ‚îî‚îÄ‚îÄ 1.3.12.1_folderReadme/
          ‚îî‚îÄ‚îÄ 1.3.12.1.a_fileMyNewProtocolReadme.md
```

### 2. Implement the Interface

```typescript
export const myNewProtocolModule: ProtocolHandshakeModule = {
  protocolIdentifier: 'my-new-protocol',
  protocolDisplayName: 'My New Protocol',
  protocolRequiresCallbackUrl: false,
  CredentialFormFieldsComponent: MyNewProtocolCredentialFormFields,
  whitepaperDocumentationText: MY_NEW_PROTOCOL_WHITEPAPER,
  executeHandshake: async (config, modelInput, dynamicInput) => {
    // Your execution logic here
  }
};
```

### 3. Register in Index

```typescript
// In 1.3.a_fileIndex.ts
import { myNewProtocolModule } from './1.3.12_folderMyNewProtocol/...';

export const PROTOCOL_HANDSHAKE_REGISTRY = {
  // ... existing protocols
  'my-new-protocol': myNewProtocolModule,
};
```

## üîí Security Notes

- **Sensitive fields** (passwords, secrets, tokens) are automatically sanitized before persistence
- **CORS**: Use the optional CORS proxy setting for browser-based API testing
- **OAuth secrets**: Never commit actual secrets to the repository
- **Template pattern**: All credentials use `YOUR_X_HERE` placeholders
